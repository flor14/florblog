The art of asking Google

3 May 2019 in HelloWorld

How do I got there?

I’ll start with an example. When I started graphing, I used ´plot()´ to generate a line graph. At that time I thought that writing code was something quite rigid: a point or a comma of more or less made nothing work. It was not conceivable for me the idea that there could be another redundant function with ´plot()´, that is to say that it would allow me to do the same graph. Why would there be 2 functions that do the same? It did not make sense. I did not know, for example, that these graphics with ´plot()´ could also do them with a function of the ggplot2 package. This preconception brought me problems when interpreting the help of forums on the Internet, which presented possible solutions many times using different functions to solve the same problem.

Reflections

Sometimes, we can write disimilar code with the same functionalities. And I do not only mean change in structures or writing different functions, if not in the way it is written. If we speak in terms of programming languages, differences in the logic of writing the code are known as “syntax differences”. Particularly there is a group of very famous packages for data science with R as a whole tidyverse with a syntax different from R base. The importance of understanding this is that it can generate confusion when interpreting solutions in Internet forums. If in Stack Overflow someone proposes the answer to a problem using R base and another person from a tidyverse package, we must bear in mind that the writing they are proposing is different, and in a certain way they talk about different routes to reach the same result . As one who speaks two dialects of the same language and says the same It is also important to consider that the packages evolve and, consequently, change over time. Sometimes, some solutions become obsolete. For example, the package * usethis * introduced improvements in the development of packages in R after its departure in 2018, but most of the tutorials before that date, do not use it. We talked about this with PabloTe

Recommendations

  • There is controversy about whether you should learn directly at tidyverse or R base. Regardless of whether one starts learning tidyverse or R base, it is important to consider this difference to interpret the forums and tutorials. If you prefer the tidyverse packages it can help you write your “tidyverse bar graph” directly in your internet browser so that the results use the tidyverse syntax.

  • As an open source of rapid evolution, none of these approaches is guaranteed, so “you learn as you go”. Regarding the case of dates, limiting searches to the most recent ones can be a good way to obtain non-obsolete code as a result.

  • If you can not find the answer in your preferred search engine, it’s a good idea to try to read about it and find terms or keywords that bring us closer to the answers we want to find.
