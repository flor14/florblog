The aim of this post is to provide a route map highlighting the resources available to generate your own tutorial and some considerations.

Hello

Last year I helped to build some free online tutorials using Ines Montani Framework. After one busy year that included the end of my PhD finally I have some time to make a contribution to the community about how to build these tutorials.

Here you have some examples of tutorials created with this framework.

Why to use Ines Montani framework?

1- Ted Laderas has written some key “pros and cons” of using this framework in the post in the RStudio education blog explaining the difference between Ines’ framework with learnr tutorial package.

2 - Ines Montani documentation is really detailed. If you find problems in building the tutorial I strongly suggest you came back to her repo.

Do you teach a class in R and have a course that want to convert to a tuturial? A presential course it is not exactly the same as a online tutorial for example in the book Teaching Tech Together, Greg Wilson. Will do you have to adapt your material previously?

Recipe for an online tutorial 📝

1. Ingredients

First of all you have to divide your course into chapters. As a general rule there are 4 or 5 chapters.

The chapters can have:

  • Slides with explanatory text associated

  • Codeblock exercises

  • Multiple choise exercises

  • Audio or video files

You can use decampr package to generate some of the basic file structures. 🤖

2. Prepare a repository

You should have an account on GitHub (it is free). If you do not know how to do this, please check this webpage

The course starter R repository is on GitHub. You can get a copy using it as a template.

Have a look to the repository structure. In this example is for a python tutorial, but it is basically the same structure.

Remember that Ines Montani framework is available for Python too.

3. Custom elements 📦

Slides with text
  • 📁 slides folder

    • The slides are markdown (.md) files. It is good idea to name the file according to the chapter number and order, for example chapter2_01.md.

    • How is the structure of the slides file? Ines described it here

Codeblock
  • 📁 exercises folder
  • Each exercise file has an associated solution file (and a ideally a test file)
  • The names for each file should have the same reference number:

The solutions and exercises are separated files, linked by the final number in the name of the file

Multiple choise exercises
  • 📁 chapters folder
    • There are not specific files for them, are generated inside each chapter file.
    • Read more about them in the documentation

⚠️ Remember to include a unique id to each multiple choise exercise!

Audio and Video

Detailed here

Chapters
  • 📁 chapters folder
    • Each chapter is a markdown (.md) file.
    • Chapter structure: you should include the name of the slides, exercises and solution files.

Do not forget 😮

  • Customize the meta.json file with the information of your course.

  • Add a profile picture and logo too! -> 📁 static folder.

⚠️ Remember include the logo in .svg format! You can convert a .png pr .jpg image to this format online

3. Cooking!🍳

Have you added all your files to you repository? Now it is time to see if it works! Remember to read the Section: Setting up Binder and checking the file install.R -> 📁 binder folder.

This can take a while! ⏰

Meanwhile you can start your account in Netlify for the deploy as Ines did, but this step could be skipped. Why to use netlify? I found this blogpost of Yihui Xie with an explanation. It is up to you!

4. Decoration 🍒🎨

  • Add a summary and a course description as introduction
  • Change logo size modifying width argument after .logo
  • Change the colors Modify theme.sass
  • Add a licence badge! Look in the section “Add legal code” after each type of licence.
